Breaking the Memory Wall : A Paradigm Shift
Minimizes data movement and latency by executing computations directly where data resides (Near-Data Processing). Maximizes AI workload throughput by combining the ultra-high bandwidth of HBM with the flexible pooling capabilities of CXL.
• Tiered Storage:
Intelligent movement of data between SRAM, HBM, and bulk DDR/SSD based on usage frequency to balance cost and speed.
• Software-Managed Cache:
Moves away from hardware-managed caches to software-controlled scratchpads for predictable data movement in AI workloads.
• SRAM Optimization:
Maximizes on-chip SRAM capacity to keep frequently used activation data close to the arithmetic logic units (ALUs).
• Coherency:
Uses the CXL protocol to maintain cache coherency between the host processor and attached memory devices.
• Scalability:
Allows dynamic allocation of memory capacity to workloads that need it most, preventing resource stranding.
• Disaggregated Memory:
Decouples memory from specific CPUs, creating a shared pool of memory resources accessible by multiple accelerators.
• Vertical Stacking:
Utilizes 3D-stacked DRAM dies (TSV technology) to drastically increase data pathways compared to traditional DDR.
• Proximity:
Places memory stacks immediately next to the compute die (GPU/NPU) via an interposer to minimize latency.
• Bandwidth Maximization:
Delivers terabyte-scale bandwidth essential for feeding data-hungry LLMs without stalling compute cores.
• Compute Offloading:
Embeds programmable computing units directly within the memory arrays to process data where it is stored.
• Energy Reduction:
Eliminates the massive energy cost of moving data back and forth between memory and the processor.
• Parallel Efficiency:
Ideal for simple, repetitive arithmetic operations (like element-wise functions) across vast datasets.
