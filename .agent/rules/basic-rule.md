## 이미지 생성 시 참고

1. "유치한 건 절대 안 돼!" (스타일 강제 명령)
   가장 먼저 AI가 쉬운 길(저품질 이미지)로 빠지지 않게 막아야 합니다.

[필수 입력 문장] "인포그래픽을 만들되, '카툰 스타일(Cartoonish)', '클립아트(Clipart)', '손그림 느낌'은 절대 사용하지 마."

"대신 무조건 **'Clean Data Visualization Style'**과 **'Modern Tech Dashboard Aesthetic'**을 적용해. 전문적이고 분석적인 느낌이 나야 해."

2. "뭘로 그릴지 내가 정해줄게" (구성 요소 강제 명령)
   AI에게 "그림을 그려라"가 아니라 **"세련된 부품을 조립해라"**고 지시해야 합니다.

[구성 요소 지정 문장] "모든 시각 요소는 다음 규칙을 따라 조립해:

아이콘: 복잡한 그림 대신 **'Thin Line Vector Icon'**만 사용해.

도형: 둥글둥글한 기본 도형 말고, **'Geometric Shapes (기하학적 도형)'**과 **'Clean Rectangles'**를 사용해.

입체감: 촌스러운 3D 효과 대신, 도형 뒤에 아주 은은한 **'Soft Drop Shadow'**를 넣어서 세련된 깊이감만 줘.

배경 요소: 복잡한 그림 배경 대신 'Subtle Circuit Lines (은은한 회로도 라인)' 패턴을 배경에 아주 연하게 깔아줘."

3. 실전! 상황별 복붙 명령어 조합
   위의 문장들을 조합해서 실제로 이렇게 명령을 내려보세요.

상황 A: 데이터를 보여줄 때 (예: 기대효과 수치)
(X) 나쁜 예: "이 수치들을 인포그래픽으로 만들어줘." (O) 좋은 예: "이 수치들을 **'Clean Data Visualization Style'**로 시각화해. 클립아트는 쓰지 마. 데이터를 'Card UI with Soft Shadow' 안에 배치하고, 각 수치 옆에는 관련된 **'Thin Line Vector Icon'**을 하나씩 넣어줘."

상황 B: 프로세스를 보여줄 때 (예: 추진 전략)
(X) 나쁜 예: "이 과정을 그림으로 그려줘." (O) 좋은 예: "이 과정을 'Modern Tech Dashboard Aesthetic' 기반의 프로세스 도식으로 만들어. 카툰 느낌은 빼고. 각 단계를 연결할 때는 단순 화살표 대신 'Connected Geometric Nodes (연결된 기하학 노드)' 스타일을 사용해줘."
