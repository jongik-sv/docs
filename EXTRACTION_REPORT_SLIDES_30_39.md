## 4. YAML 템플릿 구조

각 YAML 파일은 다음 필수 섹션을 포함합니다:

### 4.1 메타데이터 (Metadata)
```yaml
content_template:
  id: basic-{design_intent}1
  name: "{한글 이름}"
  version: "2.0"
  category: "{카테고리}"
  source: "PPT기본양식_병합_수정(선별).pptx"
  source_slide_index: {슬라이드 번호}
  extracted_at: "{ISO 8601 타임스탐프}"
```

### 4.2 디자인 메타데이터
```yaml
design_meta:
  quality_score: {0.0-10.0}           # 템플릿 품질 점수
  design_intent: {design_intent}      # 디자인 의도
  visual_balance: balanced            # 시각적 균형
  information_density: medium         # 정보 밀도
  color_scheme: "{색상 스킴}"         # 색상 체계
  grid_structure: "{그리드 구조}"     # 그리드 구조
```

### 4.3 캔버스 정보
```yaml
canvas:
  reference_width: 1920               # 기준 너비 (픽셀)
  reference_height: 1080              # 기준 높이 (픽셀)
  aspect_ratio: "16:9"                # 종횡비
```

### 4.4 주요 구성 요소 (Main Components)
```yaml
main_components:
  - name: "{요소 이름}"
    count: {개수}
    type: "{유형}"
```

### 4.5 레이아웃 정보
```yaml
layout:
  type: "{카테고리}"
  grid:
    structure: "{그리드 구조}"
    gap_percent: {여백 비율}
```

### 4.6 사용 사례 (Use For)
- 5개 이상의 구체적 사용 사례를 한글로 작성
- 실제 비즈니스 시나리오 기반

### 4.7 키워드 (Keywords)
- 검색 및 분류를 위한 7-8개의 키워드
- 한글 및 영문 조합

### 4.8 예상 프롬프트 (Expected Prompt)
- 이 템플릿을 사용할 때의 예상 입력 프롬프트
- 구체적인 사용 예시 포함

### 4.9 호환성 및 제한사항
```yaml
compatible_with:
  - "상세 데이터 포함"
  - "이미지/아이콘 추가"
  - "텍스트 내용 커스텀"
  - "색상 테마 변경"
  - "데이터 표시"

restrictions:
  - "요소 개수 변경 시 레이아웃 재조정 필요"
  - "텍스트 길이가 길 경우 폰트 사이즈 조정"
  - "이미지 추가 시 종횡비 유지"
```

### 4.10 메타데이터
```yaml
metadata:
  category: "{카테고리}"
  difficulty: "{easy|medium|high}"
  reusability: "high"
  created_date: "{YYYY-MM-DD}"
  last_updated: "{YYYY-MM-DD}"
  content_type: "{콘텐츠 타입}"
  supports_animation: false
  supports_interactivity: false
```
